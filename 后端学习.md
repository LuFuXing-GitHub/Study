## 为什么需要分层？

| 好处     | 说明                             |
| :------- | :------------------------------- |
| 职责分离 | 每层只做自己的事，易于维护       |
| 代码复用 | Service 可被多个 Controller 调用 |
| 易于测试 | 可以单独测试每层逻辑             |
| 团队协作 | 前后端分离，后端分层开发         |

```tex
controller几乎不做业务处理，更像是一个协调者，简单来说它只接收请求，返回响应，就是不同的前端接口要调哪个service函数，调完函数返回结果，就叫做返回响应

service就是实际写业务逻辑代码的地方

DAO也就是封装好的sql语句，有对数据库的基础CRUD，在service里面处理业务的时候实际上就是在跟数据库打交道，直接调用DAO里面的函数即可

DO和VO就是定义数据结构
VO是前端和后端交互的数据结构，当前端发出请求到了controller时，controller会在这里收到VO的数据结构，同时返回响应给前端的时候用的也是VO

DO是后端和数据库交互的数据结构，DAO的CRUD查库时用的数据结构就是DO的，将DO赋值之后映射到实际的数据库字段进行交互

```
